1. Mobile Device Center 가 연결되지 않는 문제
발생원인

[문제점1] "Windows Mobile Device Center를 시작하는 동안 잠시 기다려 주십시오" 이후 진행이 안 됨.

[문제점2] 에러 메시지가 발생하면서 Windows Mobile Device Center가 실행되지 않음

해결방법

[문제점1] 해결방법

  • STEP - 1. 윈도우 돋보기 기능에서 ‘로컬 서비스’라고 입력하면 ‘로컬 서비스 보기’가 표시되며 해당 서비스를 실행합니다. 또는[제어판]→ [관리도구]→[서비스]를 선택합니다. 만약 관리도구가 안 보이는 경우 보기 기준 : ‘작은 아이콘’으로 선택합니다.
  • Step-2. ‘Windows Mobile 2003 기반 장치 연결’을 선택하고 더블 클릭하여 속성 정보를 오픈 합니다.
    참고
    Creator 버전 업데이트 이후 Windows Mobile Device Center 연결하기 전에는 Windows Mobile 2003기반 장치 연결이 보이지 않습니다. IEC-Series를 개발PC와 최초 연결하는 경우에 ‘Windows Mobile 2003 기반 장치 연결’ 가 활성화 됩니다.
!Windows10 OS의 경우 OS 업데이트가 빈번하게 발생하며 이로인해 상기에 설정한 내용이 초기화 될 수 있으니 Mobile Device Center 연결이 안되시는 경우 설정 여부를 다시 확인하시고 설정이 초기화 된 경우 재설정 하시기 바랍니다.

[문제점2 해결방법]
에러 메시지가 발생하면서 Windows Mobile Device Center가 실행되지 않는 경우에는 Microsoft 홈페이지에서 Windows Mobile Device Center 버전을 다운로드 받아 설치하시기 바랍니다.

개발 PC OS 버전 명칭
32Bit Windows Mobile Device Center 6.1-32Bit 다운로드
64Bit Windows Mobile Device Center 6.1-64Bit 다운로드
2. Visual Studio 2005/2008에서 .Net Compact Framework 관련 오류
2-1. Visual Studio 2005/2008 발생원인

Creator 버전(코드명 RedStone 2)의 업데이트 이후 Visual Studio 2005/2008에서 기존 제작된 프로젝트를 열거나 신규 프로젝트를 생성하는 경우 다음과 같은 에러가 발생합니다.

해결방법
Step-1. 만약 Visual Studio 2005/2008 이 열려있다면 프로그램을 닫은 후 아래의 파일을 각각 다운로드 후 압축 해제하고 .NET Framework 가 설치된 위치에 복사합니다. 복사하는 파일은 .NET Framework 버전에 따라 다르므로 각각의 파일을 다운로드 받은 후 .NET Framework가설치 된 위치에 파일을 복사하여 주시기 바랍니다.
1. Microsoft.CompactFramework.Common.targets 파일
2. Microsoft.CompactFramework.CSharp.targets 파일
.NET Com pact Framework 2.0 버전의 경우 .NET Compact Framework 3.5 버전의 경우
C:\Windows\Microsoft.NET\Framework\v2.0.50727 C:\Windows\Microsoft.NET\Framework\v3.5
2-2. Visual Studio 2008 발생원인

Creator 버전(코드명 RedStone 2)의 업데이트 이후 Visual Studio 2008에서 기존 제작된 프로젝트를 열거나 신규 프로젝트를 생성하는 경우 다음과 같은 에러가 발생합니다.

해결방법
Step-1. 만약 Visual Studio 2008이 열려있다면 프로그램을 닫은 후 아래의 파일을 다운로드하여 압축 해제하고 아래의 파일들을 차례대로 설치합니다.
1. NetCFSetupv20.msi
2. NetCFSetupv35.msi
3. NETCFv35PowerToys.msi
!만약 해당 파일이 기존에 설치되어있다고 나오면서 설치가 안 되는 경우 기존 설치를 제거하고 재설치를 진행하면 됩니다.
